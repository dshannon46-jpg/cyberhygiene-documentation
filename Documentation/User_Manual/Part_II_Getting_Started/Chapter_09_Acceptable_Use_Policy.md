# Chapter 9: Acceptable Use Policy

## 9.1 Policy Overview

### Purpose of This Policy

This Acceptable Use Policy (AUP) defines the rules and guidelines for using the CyberHygiene Production Network systems, services, and resources.

**Objectives:**
- Protect organizational assets and data
- Ensure compliance with legal and regulatory requirements
- Maintain system security and integrity
- Define user rights and responsibilities
- Establish consequences for policy violations

**Scope:**
This policy applies to:
- All users (employees, contractors, partners)
- All systems (servers, workstations, network devices)
- All data (stored, transmitted, or processed)
- All access methods (local, remote, mobile)

**Acknowledgment Required:**
- All users must acknowledge this policy upon account creation
- Annual re-certification required
- Violations may result in account suspension or termination

### Legal Framework

**Applicable Laws and Regulations:**
- Computer Fraud and Abuse Act (CFAA)
- Electronic Communications Privacy Act (ECPA)
- NIST Special Publication 800-171
- FIPS 140-2 Standards
- State and federal privacy laws

**Organization's Rights:**
- Monitor system usage for security and compliance
- Review logs and communications if security incident suspected
- Enforce policy through technical and administrative controls
- Revoke access for policy violations

**User's Rights:**
- Reasonable privacy expectations for personal communications
- Notice before monitoring (this policy serves as notice)
- Due process for alleged violations

## 9.3 Prohibited Activities

### Strictly Forbidden Actions

#### 1. Unauthorized Access

**‚ùå PROHIBITED:**
- Accessing systems you are not authorized to use
- Attempting to bypass security controls
- Using another person's account or credentials
- Accessing files or data beyond your need-to-know
- Port scanning or network reconnaissance (unless authorized)

**Examples of Violations:**
```
‚ùå SSH to server you don't have permission to access
‚ùå Trying to guess other users' passwords
‚ùå Using sudo when not authorized
‚ùå Accessing coworker's files without permission
‚ùå Attempting to disable security software
```

**Legal Consequences:**
- May violate Computer Fraud and Abuse Act (CFAA)
- Criminal prosecution possible
- Immediate termination

#### 2. Malicious Activities

**‚ùå PROHIBITED:**
- Installing malware, viruses, or trojans
- Creating backdoors or unauthorized access methods
- Launching attacks (DoS, DDoS, phishing)
- Data destruction or sabotage
- Cryptomining on company systems

**Zero Tolerance:**
These activities result in **immediate account termination** and **law enforcement notification**.

#### 3. Unauthorized Software

**‚ùå PROHIBITED:**
- Installing software without approval
- Using unlicensed or pirated software
- Running personal servers or services
- Installing remote access tools (TeamViewer, etc.) without authorization
- Using unauthorized cloud services for work data

**Approved Software:**
- Only install software from approved repositories
- Request software through IT ticketing system
- Use organization-provided tools

#### 4. Data Theft or Exfiltration

**‚ùå PROHIBITED:**
- Copying sensitive data to personal devices
- Emailing confidential data to personal accounts
- Uploading company data to personal cloud storage
- Removing data without authorization (even if you created it)

**Before Leaving Organization:**
- All work products belong to organization
- Return all company data and devices
- Do not retain copies of company information

#### 5. Network Abuse

**‚ùå PROHIBITED:**
- Excessive bandwidth usage (streaming, torrents)
- Operating unauthorized services (web servers, file sharing)
- Bridging network segments without authorization
- Circumventing web filters or proxies
- Connecting unauthorized devices to network

#### 6. Social Engineering and Phishing

**‚ùå PROHIBITED:**
- Impersonating other users or systems
- Creating fake login pages
- Sending phishing emails (even as tests, without authorization)
- Manipulating users to reveal credentials
- Pretexting to gain unauthorized information

**Exception:** Authorized security awareness testing by security team only.

#### 7. Harassment and Inappropriate Content

**‚ùå PROHIBITED:**
- Viewing, storing, or distributing pornographic material
- Harassment, threats, or intimidation via email/messaging
- Discriminatory or hateful communications
- Gambling or illegal activities
- Creating hostile work environment

## 9.4 User Responsibilities

### Security Responsibilities

**Every User Must:**

1. **Protect Credentials**
   - Create strong passwords (15+ characters)
   - Never share passwords
   - Change password if compromised
   - Enable MFA when required

2. **Maintain System Security**
   - Keep systems updated (apply patches)
   - Run approved antivirus software
   - Lock screen when away
   - Report security issues immediately

3. **Safeguard Data**
   - Encrypt sensitive data
   - Use secure communication channels
   - Properly dispose of documents (shred)
   - Backup important work files

4. **Report Incidents**
   - Security breaches or suspected compromises
   - Lost or stolen devices
   - Malware infections
   - Suspicious emails or activities
   - Policy violations by others

**Reporting Procedure:**
```
Security Issues:
Email: security@cyberinabox.net
Subject: [SECURITY] Brief description
Include: What, when, where, who, impact

Urgent Issues:
Contact: System Administrator immediately
Phone: (See emergency contact list)
```

### Physical Security

**Workstation Security:**
- ‚úÖ Lock screen when leaving desk
- ‚úÖ Secure laptops with cable locks
- ‚úÖ Store devices in locked areas when not in use
- ‚ùå Do not leave laptops in vehicles
- ‚ùå Do not work on sensitive data in public places

**Visitors:**
- ‚úÖ Escort visitors at all times
- ‚ùå Do not allow tailgating (follow someone through door)
- ‚úÖ Ensure visitors sign in/out
- ‚úÖ Report unauthorized persons

**Clean Desk Policy:**
- ‚úÖ Lock sensitive documents in drawers
- ‚úÖ Shred documents when no longer needed
- ‚úÖ Clear desk at end of day
- ‚ùå Do not leave sticky notes with passwords

### Compliance Responsibilities

**NIST 800-171 Compliance:**
- Follow security procedures
- Complete required training
- Acknowledge policies annually
- Participate in audits and assessments

**Data Protection:**
- Handle CUI per regulations
- Report data breaches within 72 hours
- Maintain confidentiality agreements
- Follow data retention policies

**Audit Cooperation:**
- Provide information when requested
- Participate in interviews
- Maintain audit trails
- Do not delete logs or evidence

## 9.5 Consequences of Violation

### Violation Handling Process

**Investigation:**
1. Incident reported or detected
2. Initial review by security team
3. Evidence collection (logs, screenshots)
4. User interview (if appropriate)
5. Determination of violation severity

**Factors Considered:**
- Intent (accidental vs. deliberate)
- Severity (minor vs. critical)
- Frequency (first time vs. repeat)
- Impact (no harm vs. data breach)
- Cooperation (admits vs. conceals)

### Account Actions

**Temporary Suspension:**
- Access disabled pending investigation
- Typically 3-30 days
- Can be reinstated if cleared

**Permanent Termination:**
- Account deleted
- All access revoked immediately
- Data archived for investigation
- Cannot be reinstated

### Appeal Process

**If You Disagree with Disciplinary Action:**

1. **Written Appeal:**
   - Submit within 5 business days
   - Email to: HR and System Administrator
   - Include: Your version of events, supporting evidence

2. **Review:**
   - Independent review by management
   - Additional investigation if needed
   - Decision within 10 business days

3. **Final Decision:**
   - Management decision is final
   - If upheld: original action stands
   - If overturned: record expunged, access restored

### Legal Consequences

**Civil Liability:**
- Organization may seek damages for losses
- Cost recovery for investigation and remediation
- Restitution for stolen data or property

**Criminal Prosecution:**
- Computer Fraud and Abuse Act violations
- Wire fraud, identity theft
- Copyright infringement
- Trade secret theft

**Professional Impact:**
- Security clearance revocation
- Professional license suspension
- Industry blacklist
- Reference impact
